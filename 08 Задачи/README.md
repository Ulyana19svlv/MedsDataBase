# Задачи

`tasks.json` — будущий явный слой контрольных задач.

Сейчас дашборд уже умеет вычислять задачи из `follow_up_date` в медицинских событиях. Этот файл нужен для следующего шага: чтобы агент мог создавать отдельные задачи, связанные с документами, событиями и зонами наблюдения.

## Примеры задач

- контроль липидограммы;
- повторить УЗИ;
- записаться к врачу;
- взять документ на прием;
- проверить, что рекомендация врача выполнена.

## Агент задач

Команда запускается из `06 Сайт`:

```powershell
npm run agent:tasks
```

Агент читает проверенные медицинские события из `01 Члены семьи`, ищет контрольные формулировки в разделах `Что делать дальше`, `Назначения`, `Что важно отследить`, `Контроль в июле`, `Планово` и обновляет `tasks.json`.

Записи с `source_agent: "tasks-agent"` пересоздаются при каждом запуске. Ручные записи без этого поля сохраняются.
